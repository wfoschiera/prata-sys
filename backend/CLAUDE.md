# Backend — CLAUDE.md

FastAPI + SQLModel + PostgreSQL. See root `CLAUDE.md` for full project context.

## Structure

```
backend/app/
├── api/routes/      # One file per resource (users.py, clients.py, services.py, …)
├── core/            # config.py (settings), security.py (JWT/passwords), db.py
├── alembic/         # DB migrations
├── email-templates/ # MJML email templates
├── models.py        # ALL SQLModel models (DB tables + Pydantic schemas)
├── crud.py          # ALL CRUD functions
├── main.py          # FastAPI app entrypoint, router registration
└── utils.py         # Helpers (email sending, etc.)
```

## Model Conventions

- Define table models and schema models together in `models.py`
- Use SQLModel pattern: `class Foo(SQLModel, table=True)` for DB, plain `SQLModel` for schemas
- Every table model needs `id: uuid.UUID = Field(default_factory=uuid.uuid4, primary_key=True)`
- Brazilian documents: store CPF (11 digits) and CNPJ (14 digits) as plain strings; validate format in Pydantic schemas

## CRUD Conventions

- All DB operations in `crud.py` as plain functions receiving a `session: Session` argument
- **N+1 rule**: always use `selectinload()` or `joinedload()` when fetching related objects. Never access a relationship attribute that would trigger a lazy load in a loop.
- Prefer explicit queries over ORM magic for anything non-trivial

## API Route Conventions

- One router per resource, mounted in `main.py`
- Use dependency injection for `session` (`get_db`) and `current_user` (`get_current_active_user`)
- Return Pydantic schema models, never raw ORM objects
- Prefix routes: `/api/v1/clients`, `/api/v1/services`, etc.

## Roles & Auth

- Roles: `admin`, `finance`, `client` (stored on the `User` model)
- Use `Depends(get_current_active_superuser)` for admin-only endpoints
- Finance role needs a custom dependency (add when implementing)

## Testing

```bash
# From /backend
bash ../scripts/test.sh

# With coverage
bash ../scripts/test.sh --cov
```

- Tests live in `backend/tests/`
- Use pytest fixtures for DB session and test client
- **Always run tests before committing**

## Package Management

```bash
uv add <package>       # add dependency
uv sync                # install all deps
```

## Migrations

```bash
alembic revision --autogenerate -m "short description"
alembic upgrade head
```

Always review autogenerated migrations before applying.
